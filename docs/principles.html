Core Principles

The standard derives from a small set of immutable principles. They should not change between minor versions.

ID Principle Meaning
CP-1Product Before ImplementationA product must be understood before it is implemented.
CP-2AI First, Human GovernedAI performs engineering work; humans govern decisions.
CP-3Knowledge Governs CodeProduct Knowledge, not code, is the primary asset.
CP-4Architecture Governs ImplementationImplementation conforms to architecture, never the reverse.
CP-5Build Through IntentionsThe Intention is the smallest governed unit of delivery.
CP-6AI Requires Governed ContextAI never operates on unrestricted repository knowledge.
CP-7Verify Before ProgressNo implementation progresses without evidence.
CP-8Preserve Product KnowledgeVerified knowledge becomes permanent product memory.
CP-9Markdown FirstKnowledge is authored in open, version-controlled text.
CP-10Continuous EvolutionA product is never finished; each Intention strengthens it.
CP-11Product ConvergenceEvery change leaves the product equal or better; entropy falls.

This is a summary. The normative source is framework/CORE_PRINCIPLES.md. On any conflict, the specifications and framework documents in the repository prevail.
