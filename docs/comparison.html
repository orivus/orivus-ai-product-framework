What the Orivus AI Product Framework is and is not

The framework does not replace existing standards. It addresses a different layer: how products evolve when AI executes most of the engineering.

A new layer, not a replacement

Each approach below organizes a different part of software work. The framework organizes product evolution under AI execution.

Side by side

Approach Primary question What it governs Relationship to the framework
Scrum / Agile How do teams deliver incrementally? Cadence, roles, ceremonies Complementary — the framework does not prescribe team structure
DDD How do we model the domain? Bounded contexts, ubiquitous language Complementary — domain knowledge becomes part of Product Knowledge
OpenAPI How do we describe APIs? Interface contracts Complementary — API specs are one artifact type inside governed delivery
Spec-driven development How do we write specs before code? Feature specifications → implementation Overlapping intent, different scope — the framework governs the full product lifecycle, not a single feature spec
OpenSpec / change proposals How do we propose and track changes? Change documents and approval flow Overlapping intent — the framework adds verified execution, milestone evidence, and institutional knowledge retention
AI coding assistants How do we generate code faster? Output velocity Different problem — assistants optimize generation; the framework optimizes governed evolution and knowledge durability
Orivus AI Product Framework How do products evolve when AI executes engineering? Product Knowledge, Intentions, verified outcomes —
The category

AI Product Engineering

The Orivus AI Product Framework names a category — AI Product Engineering — not a replacement for Scrum, DDD, or OpenAPI. The principle underneath: Product Knowledge governs implementation.

The distinction

Not another codegen framework

Spec-driven tools ask: what should we build next? The framework asks: how does the product keep evolving without losing what it already knows?

Product Knowledge is the permanent asset. AI is the current executor. Source code is a derived artifact.
